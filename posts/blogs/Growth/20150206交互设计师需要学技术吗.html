交互设计师需要学技术吗？

大家好，今天讨论的问题是交互设计师需要学技术开发吗？毫无疑问，这是需要的。交互设计是一门集中了多门学科的新学科，起源也是因为计算机的兴起才逐渐形成这门学科。在微软，交互设计师这个概念已经逐渐模糊，渐渐成为用户体验设计师（之前一位在微软工作的实习导师跟我说），最重要是要自己学会编程写原型，可想而知，编程对于交互设计师来说也是门必要的技能之一。今天给大家带来大学期间小弟看的技术类书籍，把这些书都掌握了估计做个初级应用层级全栈开发者应该没什么问题。

先介绍 Web 篇，小弟先从 w3school （非常重要的网站）开始学习 html ， css ，然后到自学 php ，再自学了 jQuery ，期间大大小小独自开发了几个网站，再深入学习了 mysql 和如何构建站点和优化服务器，以下是小弟强烈推荐的书籍。可以按照小弟看书的顺序来学习 web 开发，做 web 开发最重要还是靠搜索，很多优秀的 html5 和 js 的资源都在网上。

1. HTML5 权威指南

2. PHP 和 MySQL Web 开发

3. jQuery 基础教程

4. JavaScript 高级程序设计

5. 响应式 Web 设计 HTML5 和 CSS3 实战

6. MySQL 必知必会

7. 高性能 MySQL

8. 深入 PHP ：面向对象、模式与实践

9. JavaScript 权威指南（当工具书就好）

10. 鸟哥的 Linux 私房菜 - 基础学习篇

11. 鸟哥的 Linux 私房菜 - 服务器架设篇

12. 构建高性能 Web 站点

再介绍客户端篇，大学期间因尝试独立做一个 android 项目来创业，期间先学习了 java ，然后学习 android 和服务器的搭建，现在也因为想做个 app 玩玩，在自学 ios 的开发，以下是小弟学习的顺序

1. Java 编程思想

2. Java 核心技术·卷 1 基础知识

3. Java 核心技术（卷 2 ）：高级特性

4. Android 4 高级编程

5. 深入理解 Android （卷 1 ）

6. Android 编程权威指南

7. Android 框架揭秘

8. Android 传感器高级编程

9. 疯狂 iOS 讲义（上）

10. 疯狂 iOS 讲义（下）

11. iOS 开发指南 从零基础到 App Store 上架

12. Swift 开发指南

最后到其他种类篇，这些书也是感兴趣才看的，重点介绍这几本

1. 数据挖掘导论完整版

2. Node.js 开发指南

3. 深入浅出 Node.js

4. Python 基础教程

5. 算法第四版

虽然从大二开始转型学习交互设计，但始终觉得最后离不开编程这工具，所以按照自己兴趣把自己想学的技术都学了，但网络、汇编、操作系统和其他底层部分的技术觉得以后自己也用不着所以就没兴趣学了，但起码没挂科 哈哈哈。想真正地把这些技能学好，最重要是实践实践再实践，希望这些对大家有帮助吧。
